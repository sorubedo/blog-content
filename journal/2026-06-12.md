**04:01**

累，但又不是那种「脑子转不动了」的累。

今天从早到晚跟 DeepSeek v4 Pro 死磕那个杀戮尖塔卡牌制作器。氛围编程这个东西很神奇——你说费脑子吧，确实不怎么费，大部分时间在等 AI 输出、看它表演、然后骂它。但你说不费吧，莫名其妙整个人就被掏空了。就像一个监考老师，坐那儿啥体力活没干，光是盯着学生别作弊，下班回家一样瘫成泥。

这就是 vibe coding 的真相：脑力消耗接近零，情绪消耗拉满。看似你在「编程」，实际上你在做一种很新的客服工作——不停地跟 AI 解释它哪里错了、为什么错了、你他妈能不能别再往那个方向猜了。DeepSeek 今天把 enum 类型丢了说成 Canvas API 兼容性问题，长篇大论几十轮，我就像一个反复纠正 ChatGPT 幻觉的绝望用户，唯一的区别是这玩意还花了我 20 块钱。

有时候真想回到古法编程——那种安静地、可控地、一步步排错的方式。至少你不会被自己雇来的 AI 气到血压飙升。

不过好消息是，微信小程序开发实践的大作业算是搞完了。这个卡牌制作器就是选题，能跑、能导出、中英双语，挺好。

但完全没法松口气——还有 Web 开发实践的大作业、软件测试的大作业在排队。这学期课表是空的，但我他妈的比上课还忙。

睡了，明天继续当 AI 的监考老师。

**04:10**

哦对，差点又忘了——上次日记说的 tray-host 项目，还是没写。明天一定。

说说最近搞的另一件事：把 qBittorrent 重新整起来了。校园网这破网，IPv6 公网不给，IPv4 一个 NAT-3 一个 NAT-4，TCP UDP 双双阵亡，打洞是不可能打洞的。这次走的内网穿透——frp 架在 VPS 上，qBit 躲后面。

但这套有个膈应的问题：所有通过 frpc 进来的用户，IP 全部显示为 127.0.0.1。peerbanhelper 直接废了——它看谁都是本地回环(带I标志的)，看到吸血雷也没法ban，血压微升，换个qBittorrent Enhanced Edition倒也能解决。

好在出站主动连接的用户 IP 显示是正常的。所以大概算是「半个公网做种」——我通过代理连 tracker，tracker 记录我 VPS 的公网 IP；别人没公网的可以通过 tracker 发现我 VPS IP 然后走 frp 隧道连上我；别人有公网的我直接出站打过去，能通。一套野路子，但确实跑起来了。

DHT 和 PEX 就惨了。来自这两条路子的用户根本不知道我的公网 IP 是多少。DHT 之前一直是 0，怀疑是容器部署的锅，改成 host 网络模式好了一阵，现在不知道为啥又归零了。懒得折腾，干脆关掉了。

P2P 真好玩，P2P 得玩。校园网再怎么围堵也挡不住一个想跑上传的人。下次写篇博客记录下这套配置。

**14:16**

我恨 systemd !!!!!!。

今天打开电脑，发现文件选择器弹不出来了。portal 没了。门户没了。一查日志，`graphical-session.target` 依赖没满足，`xdg-desktop-portal` 直接摆烂不启动。

我：？？？我这不是在图形界面里吗，屏幕亮着呢我在敲终端呢。

systemd：不，你不在。

现在的 systemd 发行版跟商量好了似的，都在逼各个 DE/WM 往 systemd 里集成。Gnome 和 KDE 早早投了，sway / niri 也给了面子写了 systemd 服务。但我用的 river——一个 Wayland 平铺合成器，社区不大，没人写这玩意。于是 systemd 就硬装瞎：你没注册 `graphical-session.target`？那你就不在图形会话里，别想用 portal。

真搞不懂把图形会话写成 systemd 单元再启动的意义在哪。以前 X11 时代一个 `startx` 搞定的，现在我得自己写 service 文件告诉 systemd「嘿我这确实有个显示器，有人在用鼠标，别闹了」。这到底是简化还是复杂化？

没办法，老老实实去把 river 的 systemd 服务写了，挂在 `graphical-session.target` 下面。真他妈的烦。

这就是 Linux 桌面的终极宿命吧——你不想折腾它，它总有办法逼你折腾。
